# 🔧 Lunch Pick 문제 해결 가이드

## ❌ "카카오맵 API가 로드되지 않았습니다!"

### 원인
카카오 개발자 콘솔에서 **플랫폼 등록**이 안 되어 있습니다.

### 해결 방법

#### 1단계: 카카오 개발자 콘솔 접속
```
https://developers.kakao.com/console/app
```

#### 2단계: 앱 선택
- "Lunch Pick" (또는 생성한 앱 이름) 클릭

#### 3단계: 플랫폼 등록
1. 왼쪽 메뉴에서 **"플랫폼"** 클릭
2. 하단 **"Web 플랫폼 등록"** 버튼 클릭
3. 사이트 도메인 입력:
   ```
   https://jsunny10.github.io
   ```
   
   ⚠️ **주의사항**:
   - `http`가 아닌 `https`
   - 끝에 `/` 없이
   - 포트 번호 없이

4. **"저장"** 버튼 클릭

#### 4단계: 확인
1. 플랫폼 목록에 `https://jsunny10.github.io` 표시되는지 확인
2. 웹사이트로 돌아가서 **강력 새로고침**:
   - Windows: `Ctrl + Shift + R` 또는 `Ctrl + F5`
   - Mac: `Cmd + Shift + R`
3. 🐛 버튼 눌러서 "✅ 카카오맵 API 로드 성공" 확인

---

## 🐛 "식당 검색: 0개 발견"

### 원인
1. 입력한 주소가 잘못됨
2. 선택한 거리가 너무 짧음
3. 해당 지역에 실제로 식당이 없음

### 해결 방법
1. **주소 재입력**: "서울시청", "강남역", "명동역" 등 명확한 주소
2. **거리 늘리기**: "도보 500m" → "차량 3km"
3. **음식 종류 늘리기**: 한식만 선택 → 여러 종류 선택
4. **다이어트 모드 끄기**: 필터가 너무 엄격할 수 있음

---

## 🗺️ "지도가 안 보여요"

### 체크리스트
- [ ] 카카오맵 API 로드 성공 (🐛 콘솔 확인)
- [ ] 식당 검색 결과 있음
- [ ] 브라우저 캐시 삭제

### 해결 방법
1. 🐛 버튼 클릭
2. "✅ 카카오맵 API 로드 성공" 확인
3. "식당 검색: X개 발견" (0개가 아닌지 확인)
4. 여전히 안 보이면 → 브라우저 강력 새로고침

## 📍 "현재 위치를 가져올 수 없습니다"

### 원인
브라우저 위치 권한 거부

### 해결 방법

#### Chrome (모바일)
1. 주소창 왼쪽 자물쇠 아이콘 탭
2. "사이트 설정"
3. "위치" → "허용"

#### Safari (iPhone)
1. 설정 > Safari > 위치
2. "허용" 선택

#### Chrome (PC)
1. 주소창 왼쪽 자물쇠/정보 아이콘 클릭
2. 위치 → 허용

---

## 💡 빠른 테스트 방법

### 1. 데모 모드로 테스트
카카오 API 없이도 작동하는지 확인:
- 아무 주소나 입력
- "추천 받기" 클릭
- "🔧 데모: ..." 식당이 나오면 페이지는 정상

### 2. 간단한 주소로 테스트
```
서울시청
강남역
명동
```

### 3. 관대한 조건으로 테스트
- 거리: 차량 5km
- 음식: 모든 종류 선택
- 별점: 전체
- 다이어트: OFF
